# SANY_ELE_RANK 异常能源使用评分算法文档

## 1. 算法目标

检测宿舍夜间（23:00 - 次日 06:00）的突发性大功率用电行为（如热水器、电吹风、火锅等），输出一个 0~100 分的**异常能源使用分数**。

**核心原则：**
- 只关心"突然峰值"，不关心整体平均用电高低
- 平稳高值（如全夜开空调 1.5 kWh/h）不应获得高分
- 突发性尖峰（如平时 0.5 突然 3.0）应重重加分
- 空调典型模式（前半夜 1.5、后半夜 0）分数不能过高

---

## 2. 输入数据

23:00 到次日 06:00，每小时的用电量，共 **7 个数据点**：

| 变量 | 时段 | 含义 |
|------|------|------|
| x1 | 23:00 - 00:00 | 该小时用电量（kWh） |
| x2 | 00:00 - 01:00 | 该小时用电量（kWh） |
| x3 | 01:00 - 02:00 | 该小时用电量（kWh） |
| x4 | 02:00 - 03:00 | 该小时用电量（kWh） |
| x5 | 03:00 - 04:00 | 该小时用电量（kWh） |
| x6 | 04:00 - 05:00 | 该小时用电量（kWh） |
| x7 | 05:00 - 06:00 | 该小时用电量（kWh） |

---

## 3. 公式推导

### 步骤 1：计算每个小时的"尖峰突出度"（spike）

尖峰突出度衡量某一小时的用电量相对于相邻小时的突出程度。

**边界点（首尾）**：与唯一相邻小时比较

```
spike_1 = max(0, x1 - x2)
spike_7 = max(0, x7 - x6)
```

**中间点（i = 2 到 6）**：与前后两小时的平均值比较

```
spike_i = max(0, x_i - (x_{i-1} + x_{i+1}) / 2)
```

**设计思路：**
- `max(0, ...)` 确保只计算正向突出（凸起），下降不算异常
- 中间点使用两侧平均值作为基准，能精确捕捉"凸起"的幅度
- 边界点只能与单侧比较，是合理的简化

### 步骤 2：求总突出度 S

```
S = spike_1 + spike_2 + spike_3 + spike_4 + spike_5 + spike_6 + spike_7
```

S 表示整晚所有尖峰突出度的累计值。

### 步骤 3：映射到 0~100 分

```
分数 = min(100, S * 25)
```

乘以 25 的含义：总突出度达到 4 kWh 时即封顶 100 分。

---

## 4. 参数说明

| 参数 | 当前值 | 作用 | 调整建议 |
|------|--------|------|----------|
| 映射系数 | 25 | 控制敏感度，越大越敏感 | 增大 -> 更敏感（如 30），减小 -> 更宽松（如 20） |
| 满分阈值 | 4 kWh (= 100/25) | S 达到此值即 100 分 | 由映射系数决定 |
| 数据点数 | 7 | 23:00-06:00 每小时 | 固定值 |

---

## 5. 典型场景验证

### 场景 1：空调前半夜开、后半夜关

```
输入：[1.5, 1.5, 1.5, 0, 0, 0, 0]

spike_1 = max(0, 1.5 - 1.5) = 0
spike_2 = max(0, 1.5 - (1.5+1.5)/2) = 0
spike_3 = max(0, 1.5 - (1.5+0)/2) = max(0, 1.5 - 0.75) = 0.75
spike_4 = max(0, 0 - (1.5+0)/2) = max(0, -0.75) = 0
spike_5 = max(0, 0 - (0+0)/2) = 0
spike_6 = max(0, 0 - (0+0)/2) = 0
spike_7 = max(0, 0 - 0) = 0

S = 0.75
分数 = min(100, 0.75 * 25) = 18.75 分

结论：不算太高，符合预期（空调正常关闭过渡）
```

### 场景 2：平稳低用电

```
输入：[0.1, 0.3, 0.1, 0.2, 0.1, 0, 0]

spike_1 = max(0, 0.1 - 0.3) = 0
spike_2 = max(0, 0.3 - (0.1+0.1)/2) = max(0, 0.3 - 0.1) = 0.2
spike_3 = max(0, 0.1 - (0.3+0.2)/2) = max(0, 0.1 - 0.25) = 0
spike_4 = max(0, 0.2 - (0.1+0.1)/2) = max(0, 0.2 - 0.1) = 0.1
spike_5 = max(0, 0.1 - (0.2+0)/2) = max(0, 0.1 - 0.1) = 0
spike_6 = max(0, 0 - (0.1+0)/2) = 0
spike_7 = max(0, 0 - 0) = 0

S = 0.3
分数 = min(100, 0.3 * 25) = 7.5 分

结论：几乎不加分，正常待机用电
```

### 场景 3：突发大功率用电

```
输入：[0.5, 0.5, 3.0, 0.5, 0.5, 0.3, 0.1]

spike_1 = max(0, 0.5 - 0.5) = 0
spike_2 = max(0, 0.5 - (0.5+3.0)/2) = max(0, 0.5 - 1.75) = 0
spike_3 = max(0, 3.0 - (0.5+0.5)/2) = max(0, 3.0 - 0.5) = 2.5
spike_4 = max(0, 0.5 - (3.0+0.5)/2) = max(0, 0.5 - 1.75) = 0
spike_5 = max(0, 0.5 - (0.5+0.3)/2) = max(0, 0.5 - 0.4) = 0.1
spike_6 = max(0, 0.3 - (0.5+0.1)/2) = max(0, 0.3 - 0.3) = 0
spike_7 = max(0, 0.1 - 0.3) = 0

S = 2.6
分数 = min(100, 2.6 * 25) = 65 分

结论：重重加分，成功检测到 01:00-02:00 的突发用电
```

### 场景 4：小幅异常 + 空调

```
输入：[1.5, 2.5, 1.5, 0, 0.2, 0, 0]

spike_1 = max(0, 1.5 - 2.5) = 0
spike_2 = max(0, 2.5 - (1.5+1.5)/2) = max(0, 2.5 - 1.5) = 1.0
spike_3 = max(0, 1.5 - (2.5+0)/2) = max(0, 1.5 - 1.25) = 0.25
spike_4 = max(0, 0 - (1.5+0.2)/2) = 0
spike_5 = max(0, 0.2 - (0+0)/2) = 0.2
spike_6 = max(0, 0 - (0.2+0)/2) = 0
spike_7 = max(0, 0 - 0) = 0

S = 1.45
分数 = min(100, 1.45 * 25) = 36.25 分

结论：明显加分，检测到 00:00-01:00 的异常峰值
```

### 场景 5：极端大功率

```
输入：[0.2, 0.1, 0.2, 4.0, 0.3, 0.1, 0.1]

spike_1 = max(0, 0.2 - 0.1) = 0.1
spike_2 = max(0, 0.1 - (0.2+0.2)/2) = 0
spike_3 = max(0, 0.2 - (0.1+4.0)/2) = 0
spike_4 = max(0, 4.0 - (0.2+0.3)/2) = max(0, 4.0 - 0.25) = 3.75
spike_5 = max(0, 0.3 - (4.0+0.1)/2) = 0
spike_6 = max(0, 0.1 - (0.3+0.1)/2) = 0
spike_7 = max(0, 0.1 - 0.1) = 0

S = 3.85
分数 = min(100, 3.85 * 25) = 96.25 分

结论：接近满分，极端用电行为被严厉标记
```

|------|----------|------|------|

## 7. 可调参数指南

如果需要调整算法敏感度：

- **想更敏感**（小异常也要抓）：将映射系数从 25 调高到 30~35
- **想更宽松**（只抓明显异常）：将映射系数从 25 调低到 15~20
- **满分门槛对照**：
  - 系数 20 -> 5 kWh 突出度满分
  - 系数 25 -> 4 kWh 突出度满分（当前）
  - 系数 30 -> 3.33 kWh 突出度满分
  - 系数 35 -> 2.86 kWh 突出度满分
